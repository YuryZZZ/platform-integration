# /sync-lovable

> Prerequisites: Ensure all your local backend/Supabase logic is committed so Lovable doesn't overwrite it.

1. Fetch latest changes from GitHub (where Lovable writes its code):
   ```powershell
   git fetch origin
   ```

2. Check for potential merge conflicts between Antigravity backend logic and Lovable UI components:
   ```powershell
   git status
   ```

3. Execute the strict pull:
   ```powershell
   git pull origin main --rebase
   ```

4. Verify the App Router Integrity:
   - Lovable operates purely in `src/components/` and `src/pages/`.
   - Ensure Lovable hasn't accidentally modified `src/components/blocks/` (Nexus components) or `.env.local`.
   - Run `npm run build` verify to confirm no type errors were introduced by Lovable.

5. Visual Check:
   - Use `browser_subagent` to view the newly ingested Lovable components on `localhost:3000`.
