# Portable Framework Template

This file defines the final reusable shape of the Antigravity project template.

## Goal

A new machine should be able to clone `platform-integration`, run `start.bat`, open the generated project, run `init.ps1`, and proceed without editing any repo-owned machine-specific paths.

## Required Template Guarantees

- `start.bat` must work when the repo is copied anywhere on disk.
- `starter-kit/` must be self-contained after copy.
- `starter-kit/init.ps1` must exist in the generated project root.
- `reference/` must contain the platform docs needed for Antigravity, Firebase, GitHub, Jules, and Stitch.
- `configs/*.reference` files must use placeholders, never a specific username or absolute personal path.
- Secrets must stay in `.env.local`, Antigravity settings, or local MCP config only.

## Final Bootstrap Flow

1. Clone or download `platform-integration` on the new PC.
2. Run `start.bat` from the repo root.
3. Enter the project name and output folder.
4. Open the generated project folder in Antigravity.
5. Run `./init.ps1`.
6. Fill `.env.local` and `reference/configs/mcp_config.json.reference` with machine-specific values.
7. Build, validate, and deploy.

## Generated Project Layout

```text
my-project/
|-- init.ps1
|-- README.md
|-- firebase.json
|-- .firebaserc
|-- .agents/
|-- .github/
|-- docs/
|-- functions/
|-- specs/
|-- scripts/
|-- ui/
|-- web/
`-- reference/
    |-- README.md
    |-- NEW_PROJECT_GUIDE.md
    |-- AUTH_FLOW.md
    |-- JULES_INTEGRATION.md
    |-- STITCH_DESIGN.md
    |-- VERCEL_DEPLOYMENT.md
    `-- configs/
```

## Per-PC Values That Must Be Replaced Locally

- `YOUR_USER`
- `YOUR_GCP_PROJECT_ID`
- `YOUR_BIGQUERY_LOCATION`
- `YOUR_STITCH_API_KEY`
- `YOUR_PERPLEXITY_API_KEY`
- GitHub PAT in Antigravity MCP config
- Firebase/GCP auth generated by `firebase login` and `gcloud auth application-default login`

## Validation Checklist

- `start.bat` copies the starter kit without referencing the original author machine.
- The generated folder contains `init.ps1` at the root.
- The generated folder contains `reference/PORTABLE_FRAMEWORK_TEMPLATE.md`.
- `starter-kit/README.md` instructions match the actual bootstrap flow.
- `configs/mcp_config.json.reference` contains placeholders only.
- No checked-in file requires `C:\Users\yuryz\...` to function.

## Recommended First Commands On A New PC

```powershell
gcloud auth login
gcloud auth application-default login
firebase login
cd C:\path\to\generated-project
.\init.ps1
cd web
npm install
npm run build
cd ..\ui
npm install
npx playwright install chromium
```

## Definition Of Done

The template is done when a second Windows PC can bootstrap a new project without editing repository code before first run.
